есть задание по теме "имитационное моделирование вко" (военная тематика)

нужно написать 3 скрипта на bash (для ПРО, РЛС и ЗРДН)

сначала объясню, как написать скрипт для ПРО


в директории /tmp/GenTargets/Targets находятся файлики (пример названия одного файла - 21e904ef2f670ac41c)
в этом названии первые 12 символов - случайная строка, оставшиеся 6 символов - ID цели
каждый такой файлик содержит примерно такое: "X4273243,Y3901432" - это координата цели (в метрах)
диапазон координат: для X - от 0 до 13000000 метров, для Y - от 0 до 9000000 метров
одна из та же цель имеет один и тот же ID и при создании нового файла имеет новые координаты (она движется)
новые файлики для каждой цели генерируются каждую секунду (это значение в константах)
ПРО расположена по координатам (которые задаются в командной строке) 
ПРО должна захватывать цели в радиусе своего действия (радиус задается в командной строке)

есть три состояния, в которых может находится цель
1. если этой цели нет в состоянии 1 или в состояни 2, то это новая цель.
новая цель должна находится в радиусе действия ПРО и нужно сохранить ее изначальные координаты
как проверить, что цель находится в радусе действия? Нужно посчитать расстояние до нее: (x0 - x)^2 + (y0 - y)^2 < R^2
где x0, y0 - координаты ПРО, x, y - координаты цели, R - радиус действия ПРО. Если неравенство выполняется, то цель находится в радиусе действия
2. если цель находится в состоянии 1, то для этой цели нужно посчитать скорость (разница начальных и координат в следующий момен твремени, поделенные на 1 секунду)
в зависимости от скорости, цель разделяется на
- ББ БР - 8000-10000 м/c     
- Крылатой ракеты - 250-1000 м/c
- Самолета - 50-250 м/c
Если это новая цель, то не важно, какая цель, нужно записать информацию в лог файл temp/pro1.log вида
"{время} ПРО Обнаружена цель {тип цели} с ID:{ID цели} X={X} Y={Y}"

3. Исходя из скорости, можно определить - это цель ББ БР? Если цель ББ БР, то цель такая цель переходит в состояние 3
Если цель ББ БР, то нужно уничтожить цель, для этого надо в /tmp/GenTargets/Destroy положить файлик с названием ID цели без содержания
Если цель ББ БР, то нужно записать информацию в лог файл temp/pro1.log вида "{время} ПРО выстрел по цели ID:{ID цели} X={X} Y={Y}"

скрипт должен читать 30 последних файлов из директории /tmp/GenTargets/Targets
если цель находится в состоянии 3, однако генерируется новый файл с ее ID, то значит мы промахнулись
в таком случае нужно записать информацию в лог файл temp/pro1.log вида "{время} ПРО промах по цели ID:{ID цели} X={X} Y={Y}"
а затем "{время} ПРО повторный выстрел по цели ID:{ID цели} X={X} Y={Y}"

как хранить состояния? создавай для каждого состояния свой файлик вида: pro1_state{number}.log (pro1_state1.log, pro1_state2.log, pro1_state3.log)
- в pro1_state1.log нужно хранить id цели и ее изначальные координаты
- в pro1_state2.log нужно хранить id цели, ее скорость и тип этой цели
- в pro1_state3.log нужно хранить id цели

через аргументы командной строки задаются радиус действия, координаты x и y ПРО
